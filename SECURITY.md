# Security Policy

## Supported Versions

The latest version will always receive security updates. Older versions, at this time, will not be
maintained. New versions are expected to be backward compatible whenever possible.

## Reporting a Vulnerability

Please report vulnerabilities in the [Security Advisory Form](https://github.com/josolanes/ContainrBot/security/advisories/new)
per the [GitHub docs](https://docs.github.com/en/code-security/how-tos/report-and-fix-vulnerabilities/privately-reporting-a-security-vulnerability)

Vulnerabilities will always take highest priority and we try to respond same day to indicate we received
the Vulnerability Advisory

Vulnerability Advisory communications:
1. We will respond within 3 days to allow time to research the Advisory before indicating
if it's been accepted or declined
    1. If accepted, resolving the vulnerability will stay our highest priority and a
fix will be implemented as soon as possible (varies based on complexity)
        1. Once a fix has been implemented you will receive an additional update when
the new release has been created
    2. If declined, the Advisory will be closed and no futher communication will occur
        1. If you feel this is in error, please report again referencing the previous
Advisory and indicating an error
